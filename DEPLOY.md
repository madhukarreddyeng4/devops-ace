# CloudPrep — Deployment Guide

This is a Vite + React frontend backed by **Lovable Cloud (managed Supabase)** plus three Razorpay edge functions.

You have two paths:

1. **Quick path** — Dockerize only the frontend, keep backend on Lovable Cloud (works in minutes).
2. **Full self-host** — Run Supabase yourself on EC2 alongside the frontend (more work, full ownership).

---

## Path 1 — Frontend on EC2, backend on Lovable Cloud (recommended)

### 1. Provision EC2
- Amazon Linux 2023 or Ubuntu 22.04, `t3.small` minimum.
- Security group: inbound `22` (your IP), `80`, `443` from `0.0.0.0/0`.
- Elastic IP attached.

### 2. Install Docker
```bash
# Amazon Linux 2023
sudo dnf install -y docker
sudo systemctl enable --now docker
sudo usermod -aG docker ec2-user

# Ubuntu
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker ubuntu

# Compose plugin
sudo curl -SL https://github.com/docker/compose/releases/latest/download/docker-compose-linux-x86_64 \
  -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```
Re-login so the `docker` group takes effect.

### 3. Pull your code
```bash
git clone https://github.com/YOUR_USER/YOUR_REPO.git cloudprep
cd cloudprep
cp .env.example .env
# Edit .env with your Lovable Cloud project's URL, anon key, project id
# (find them in Lovable → Cloud → Settings → API)
```

### 4. Build & run
```bash
docker compose up -d --build
docker compose logs -f
```
Visit `http://YOUR_EC2_PUBLIC_IP` — the app should load.

### 5. Razorpay webhook
In your Razorpay dashboard, add a webhook pointing to:
```
https://<your-cloud-project>.functions.supabase.co/razorpay-webhook
```
Subscribe to events: `payment.captured`, `payment.failed`. Set the secret to whatever you stored in `RAZORPAY_WEBHOOK_SECRET`.

### 6. HTTPS with Caddy (one-liner)
Replace `cloudprep.example.com` with your domain (point its A record to the EC2 IP):
```bash
docker run -d --name caddy --restart unless-stopped \
  --network host \
  -v caddy_data:/data -v caddy_config:/config \
  caddy:2 caddy reverse-proxy --from cloudprep.example.com --to :80
```
Caddy auto-provisions Let's Encrypt certs. (You'll need to stop the existing nginx container or change its port mapping to `8080:80` first.)

Or use **certbot + nginx** if you prefer.

### 7. Updates
```bash
git pull
docker compose up -d --build
```

## Path 2 — Self-hosted Supabase on the same EC2

Lovable Cloud is managed Supabase. The application code (frontend + edge functions + SQL) is portable — you can run it on your own Supabase install.

### 1. Bigger EC2
Self-hosted Supabase runs ~10 containers (Postgres, GoTrue, PostgREST, Realtime, Storage, Kong, Studio, Edge Runtime, Meta, Imgproxy). Use **`t3.medium` or larger** (4GB RAM minimum, 8GB recommended). Add a 30GB+ EBS volume for Postgres data.

### 2. Clone the official self-hosting repo
```bash
git clone --depth 1 https://github.com/supabase/supabase
cd supabase/docker
cp .env.example .env
```
Generate strong secrets (replace defaults) for: `POSTGRES_PASSWORD`, `JWT_SECRET` (min 32 chars), `ANON_KEY`, `SERVICE_ROLE_KEY`, `DASHBOARD_USERNAME`, `DASHBOARD_PASSWORD`, `SECRET_KEY_BASE`, `VAULT_ENC_KEY`.
Use the JWT generator: <https://supabase.com/docs/guides/self-hosting/docker#generate-api-keys>

### 3. Bring it up
```bash
docker compose pull
docker compose up -d
```
Studio is on `http://EC2_IP:3000` (basic auth from `.env`). Postgres on `5432` (lock down via SG).

### 4. Migrate the schema
From this repo, the SQL that built your database lives in `supabase/migrations/*.sql` (auto-generated by Lovable). Apply them in order:
```bash
# from your laptop or the EC2
PGPASSWORD=$POSTGRES_PASSWORD psql \
  -h EC2_IP -U postgres -d postgres \
  -f supabase/migrations/<timestamp>_initial.sql
```
(Optional) Re-run the seed inserts in `supabase/seed.sql` if you have one, or use the admin panel to seed via UI.

### 5. Deploy edge functions
Self-hosted Supabase ships an Edge Runtime container. Mount your functions folder and restart:
```bash
# In supabase/docker/docker-compose.yml find the `functions` service.
# Mount your repo's supabase/functions:
volumes:
  - ../../cloudprep/supabase/functions:/home/deno/functions:Z
```
Then set the Razorpay secrets in the `functions` service env block:
```yaml
environment:
  RAZORPAY_KEY_ID: rzp_xxx
  RAZORPAY_KEY_SECRET: yyy
  RAZORPAY_WEBHOOK_SECRET: zzz
  SUPABASE_URL: http://kong:8000
  SUPABASE_ANON_KEY: <your generated anon key>
  SUPABASE_SERVICE_ROLE_KEY: <your generated service key>
```
Restart: `docker compose up -d functions`. Functions will be available at `http://EC2_IP:8000/functions/v1/<name>`.

### 6. Point the frontend at self-hosted Supabase
Edit `.env` in the `cloudprep` directory:
```
VITE_SUPABASE_URL=http://EC2_IP:8000      # or https://api.yourdomain.com
VITE_SUPABASE_PUBLISHABLE_KEY=<the ANON_KEY you generated>
VITE_SUPABASE_PROJECT_ID=local
```
Rebuild: `docker compose up -d --build`.

### 7. HTTPS for the API too
Put Caddy or nginx in front of port 8000 with TLS. Update `VITE_SUPABASE_URL` to the HTTPS URL and rebuild.

### 8. Razorpay webhook (self-hosted)
Update the webhook URL in Razorpay dashboard to:
```
https://api.yourdomain.com/functions/v1/razorpay-webhook
```

### 9. Backups
- **Postgres:** `pg_dump` on a daily cron, push to S3.
- **EBS snapshots:** weekly via Data Lifecycle Manager.

## Day-2 Operations

| Task | Command |
|---|---|
| View logs | `docker compose logs -f web` |
| Restart | `docker compose restart web` |
| Update | `git pull && docker compose up -d --build` |
| Shell in | `docker exec -it cloudprep-web sh` |
| Free space | `docker system prune -af` |

## Razorpay setup checklist
1. Razorpay dashboard → Settings → API Keys → generate **Test mode** keys first. Add to Lovable Cloud secrets (or self-hosted env): `RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET`.
2. Settings → Webhooks → add endpoint pointing to your `razorpay-webhook` function URL. Subscribe to `payment.captured` and `payment.failed`. Set a strong secret → store as `RAZORPAY_WEBHOOK_SECRET`.
3. Test a payment in test mode (use `4111 1111 1111 1111` card or test UPI).
4. When ready, switch to **Live mode** keys in Razorpay and update the secrets.

## Making yourself an admin
After signing up via the app, run this in your DB (Lovable Cloud SQL editor or self-hosted psql):
```sql
INSERT INTO public.user_roles (user_id, role)
SELECT id, 'admin' FROM auth.users WHERE email = 'you@example.com';
```
Refresh the app — you'll see the **Admin** link in the navbar.
